# 样品信息登记页面改 Excel 模板式布局方案

> 项目：workload-tool-rust v0.4.25
> 生成日期：2026-07-09
> 用途：将当前「样品信息登记」从单列表单改为横向 Excel 模板式多行录入

---

## TL;DR

将 `SampleInfoEntry.tsx` 的登记表单从当前的「单列垂直表单」改为类似 Excel 模板纸的「横向表格多行录入」，支持一次登记多个样品；顶部保留检测类型/送样时间/状态公共信息，下方表格每行一个样品，包含：序号、送样人、所属部门、所属项目、样品批号、样品主要成分、注意事项、送样数量。若需保存「所属部门」，需后端 `sample_info_records` 增加 `department` 或 `division_id` 字段。

---

## 一、当前布局分析

当前 `SampleInfoEntry.tsx` 登记表单（L145-L192）为：
- 顶部标题 + 检测类型（`dt`）+ 序号自动生成 + 状态 Chip（待检测）
- 一个 `Paper` 内用 `Grid container spacing={2}` 排成 2 列：
  - 样品批号 / 实验室/车间
  - 送样人 / 所属项目
  - 送样时间 / 序号
  - 检测类型 / 空
  - 样品主要成分（占满 1 行）
  - 注意事项（占满 1 行）
- 提交/重置按钮在右下角

**问题**：一次只能录入一个样品，字段占满整个 Paper，填写长表单时效率低，且与纸质 Excel 模板差异大。

---

## 二、目标布局设计（Excel 模板式）

参考用户提供的 Excel 截图，目标布局如下：

```
┌────────────────────────────────────────────────────────────────────┐
│  ← 样品信息登记                    检测类型：icp  │  待检测        │
├────────────────────────────────────────────────────────────────────┤
│  送样时间：2026/07/09 14:04          序号：自动生成                   │
├────────────────────────────────────────────────────────────────────┤
│ 序号 │ 送样人 │ 所属部门 │ 所属项目 │ 送样数量 │ 样品批号 │ 样品主要成分 │ 注意事项 │
├──────┼────────┼──────────┼──────────┼──────────┼──────────┼──────────────┼──────────┤
│  1   │ 张三   │ 液相     │ A001     │    1     │ 20260709 │ 甲醇/乙醇    │ 避光     │
│  2   │ 张三   │ 液相     │ A001     │    1     │ 20260710 │ 乙醇/水      │ 避光     │
│  3   │        │          │          │          │          │              │          │
├────────────────────────────────────────────────────────────────────┤
│  备注/注意事项（整单）：                                             │
└────────────────────────────────────────────────────────────────────┘
                      [重置]  [添加行]  [删除选中行]  [提交登记]
```

### 字段映射

| 当前字段 | 新布局位置 | 说明 |
|---------|-----------|------|
| 检测类型 | 顶部公共信息 | 从 `dt` 取得，只读 |
| 送样时间 | 顶部公共信息 | 整单统一，默认当前时间 |
| 序号 | 每行自动生成 | 后端自动，前端只读 |
| 送样人 | 每行输入 | 可每行不同 |
| 所属部门 | 每行输入（新增） | 需要后端新增字段 |
| 所属项目 | 每行输入 | 可每行不同 |
| 送样数量 | 每行输入 | 控制本行样品数量，可自动展开多行 |
| 样品批号 | 每行输入 | 可每行不同 |
| 样品主要成分 | 每行输入 | 可每行不同 |
| 注意事项 | 每行输入（或整单） | 用户截图里每行有注意事项，同时顶部也可保留整单备注 |
| 实验室/车间 | 每行输入 | 原字段保留，但放到每行 |

## 三、交互设计

### 1. 顶部公共信息区
- 左侧：返回箭头 + 标题「样品信息登记」
- 右侧：
  - 检测类型：`dt` 对应的 label（如 `icp`）
  - 状态 Chip：默认「待检测」
  - 送样时间：`<input type="datetime-local">`，默认当前时间
  - 整单备注：可折叠的 TextField（可选）

### 2. 样品表格
- 表头：序号、送样人、所属部门、所属项目、送样数量、样品批号、样品主要成分、实验室/车间、注意事项
- 每行一个 `TextField` 输入框，宽度紧凑，类似 Excel 单元格
- 底部行始终有一个空行，输入任一字段后自动追加新空行（或点击「添加行」）
- 行左侧增加 checkbox，选中后点击「删除选中行」可删除
- 序号：前端临时显示 `1,2,3...`，提交时由后端自动分配真实序号

### 3. 提交行为
- 前端校验：每行至少填写「样品批号」和「样品主要成分」才允许提交
- 提交时，把每一行拆成一条 `sample_info_records` 记录
- 提交后清空表格，刷新下方记录列表

### 4. 与现有记录列表的关系
- 记录列表（部分 B）保持当前表格/卡片式，不动
- 新布局只替换部分 A 的登记表单

## 四、后端影响（需要确认）

### 4.1 必须改动
若需要保存「所属部门」，则：
- `sample_info_records` 表增加 `division_id` 或 `department_name` 字段
- `SampleInfoCreate` / `SampleInfoUpdate` 增加字段
- `sample_info_repo.rs` 的 INSERT/UPDATE 增加字段
- `frontend/src/types/index.ts` 的 `SampleInfoRecord` 增加字段
- 前端 API `createSampleInfo` 的 payload 增加字段

### 4.2 可选不动
若本次仅做前端 UI 展示，「所属部门」列可只读显示为「未分配」或从实验室反推，不实际保存。
