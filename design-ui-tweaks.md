## 需求 2：所有管理卡片内的 chip 删除 × 去掉

### 意图理解

研发项目管理卡片内关联的实验室/方法标签（Chip）右侧有 `×` 删除按钮，用户希望去掉这些删除按钮。同时，检测方法管理卡片内关联的类型标签也有 `×` 删除按钮，一并去掉。

### 影响范围（仅涉及 ManagePage.tsx）

**位置 A：研发项目管理卡片** — `ManagePage.tsx` L470-L505

```tsx
// L471-487: 关联实验室标签
{(p.lab_names || []).map(n => (
  <Chip
    key={n}
    label={n}
    size="small"
    onDelete={() => { ... }}   // ← 去除 onDelete
    sx={{ borderRadius: '10px', ... }}
  />
))}
// L488-505: 关联检测方法标签
{(p.method_names || []).map(n => (
  <Chip
    key={n}
    label={n}
    size="small"
    color="info"
    onDelete={() => { ... }}   // ← 去除 onDelete
    sx={{ borderRadius: '10px', ... }}
  />
))}
```

**位置 B：检测方法管理卡片** — `ManagePage.tsx` L708-L726

```tsx
// L710-725: 类型归标签
{(m.type_names || []).map(n => (
  <Chip
    key={n}
    label={n}
    size="small"
    onDelete={() => { ... }}   // ← 去除 onDelete
    sx={{ borderRadius: '10px', ... }}
  />
))}
```

### 改动方式

对上述三处 Chip，**删除 `onDelete` prop** 即可。删除后 Chip 自动不再显示 `×` 图标，变为只读标签样式。代码中 `onDelete` 对应的回调函数体（异步更新 API）建议保留注释/保留函数声明但不绑定（或直接整体移除 prop），以保持后续复用可能性。

### 设计图文字描述

- **研发项目管理卡片**：在卡片底部，实验室标签（蓝色背景）和检测方法标签（info 蓝色）均为纯文字 Chip，无右侧 `×` 图标
- **检测方法管理卡片**：底部类型标签同样为纯文字 Chip，无 `×` 图标
- 上述 Chip 的点击行为移除，但仍可通过编辑按钮进入编辑弹窗管理关联关系

### 不修改的 Chip（确认不需要改动）

- `DivisionChips.tsx` 中的事业部筛选 Chip（无 onDelete，仅点击筛选）
- `StatsCards.tsx` 中的统计卡片 Chip（无 onDelete）
- `SampleInfoHome.tsx` / `SampleInfoEntry.tsx` 中的状态 Chip（无 onDelete）
- `StatsPage.tsx` / `SampleStatsPage.tsx` 中的表格内 Chip（无 onDelete）
- `ManagePage.tsx` 中实验室一览弹窗内的「分析检测/研发送样」Chip（clickable 切换，非 onDelete）

## 需求 4：样品信息登记记录查询改成统计页卡片式

### 当前结构分析

**管理页样品信息登记管理（ManagePage.tsx）内**：

当前"记录查询"区域（L1063-L1162）结构为：
- 筛选条（L1066-L1087）：7 个筛选字段——起始日期、截止日期、送样人、实验室、项目、检测类型（下拉）、状态（下拉）+ 查询/重置按钮
- 表格（L1089-L1155）：10 列——序号、批号、送样人、实验室、项目、检测类型、状态、检测时间、主要成分、操作（编辑/删除按钮）
- 分页（L1156-L1161）：上一页/下一页
- 独立统计区域（L1164-L1213）在下方：仅用 Chip 显示各维度计数，布局简单

**期望改造方向**：

参照 `StatsPage.tsx` 分析检测统计页的布局风格：
1. **顶部指标卡片**（参照 StatsCards 组件）：总数、今日新增、本月新增、各状态计数等
2. **日期筛选 + 导出按钮**（参照 StatsPage 的 DateRangePicker + 导出 Excel）
3. **分类统计卡片**（参照 StatsPage 的 STAT_CARDS 网格布局）

### 改造方案

#### 步骤 1：提取独立统计页为组件或独立页面

当前"记录查询"在 ManagePage 的 `tb === 'sampleinfo'` 分支内。建议：

**方案 A（推荐）：创建独立统计页面** `SampleInfoStatsPage.tsx`
- 新建 `frontend/src/pages/SampleInfoStatsPage.tsx`
- 在路由表添加 `/manage/sampleinfo/stats` 或类似路由
- 管理页内提供入口按钮

**方案 B（轻量）：在 ManagePage 内部重构**
- 用单独的 `Paper` + `Grid` 重构 L1063-L1213 区域
- 引入 `StatsCards` 组件（需适配 `SampleInfoRecord` 类型）
- 增加导出 Excel 功能

#### 步骤 2：新建统计数据接口

当前 `getSampleInfoStats` API 已存在（ManagePage L200），返回 `siStats` 包含：
```typescript
{
  total: number;
  by_status: Array<{ name: string; count: number }>;
  by_type: Array<{ type_key: string; label: string; count: number }>;
  by_lab: Array<{ name: string; count: number }>;
  by_project: Array<{ name: string; count: number }>;
  by_user: Array<{ name: string; count: number }>;
  by_month: Array<{ month: string; count: number }>;
}
```

这些数据可直接映射到顶部指标卡片（如：total → 总记录数，by_status → 状态分布卡片）。

#### 步骤 3：UI 布局设计

### 设计图文字描述

样品信息统计页借鉴"分析检测统计"页布局：
