# v0.4.32 更新说明

> **版本**: 0.4.32 | **日期**: 2026-07-10 | **基线**: v0.4.31

## 新增功能：用户分级（角色）+ 入口可见性后台可自定义

### 1. 角色管理（管理员后台）
- 新增「角色管理」入口（管理页 → 角色管理，仅管理员可进入）
- 支持角色的增 / 删 / 改，以及权限矩阵（按分组勾选可访问的导航入口）
- 预置 5 个示例角色，管理员可在后台自由调整权限点：
  - **系统管理员**：拥有全部权限（`*`）
  - **分析检测员**：分析检测入口 + 研发项目管理 / 实验室 / 检测方法 / 回收站 / 审计 / 帮助
  - **研发送样员**：研发送样入口 + 样品信息登记管理 / 帮助
  - **样品登记员**：样品信息登记入口 + 样品信息登记管理 / 帮助
  - **查看者**：三个门户入口 + 帮助（只读）
- 系统内置角色（`is_system=1`）禁止删除，避免误删导致权限丢失

### 2. 用户分配角色
- 用户管理卡片中新增「角色」下拉，管理员可为每个用户分配角色
- 用户在「用户管理」中可被停用 / 启用、重置角色

### 3. 入口可见性（按角色自定义）
- 每个角色拥有一组「权限点」，对应系统内的导航入口
- 管理员在后台自定义每个角色能看到哪些入口，保存后立即生效
- 权限点分为两类：
  - **门户入口**：`entry:sample`（研发送样）、`entry:workload`（分析检测）、`entry:sample-info`（样品信息登记）
  - **系统管理**：`manage:projects` / `manage:groups` / `manage:divisions` / `manage:methods` / `manage:trash` / `manage:audit` / `manage:backup` / `manage:help` / `manage:sampleinfo` / `manage:users` / `manage:roles`
- 前端按当前用户所属角色的权限点自动显示 / 隐藏：
  - 首页三张功能入口卡片（无权限且非管理员则隐藏）
  - 顶部「管理」按钮（管理员或拥有任一 `manage:*` 权限可见）
  - 管理页各子卡片（按 `manage:<key>` 门控）

### 4. 鉴权与数据
- 登录 JWT 载荷新增 `role_id` 与 `permissions`；管理员恒为 `["*"]`，非管理员从库加载所属角色权限点
- 后端 `roles` / `role_permissions` 新表，`users` 表新增 `role_id` 关联；老库升级自动兼容（列已存在则忽略，种子角色按名称幂等写入）

## 升级与部署
- 版本隔离：本版本为独立的 `v0.4.32` 文件夹，不在旧版本上原地修改
- Windows 安装包固定 AppId，可直接覆盖安装上个版本
- Docker 镜像随 `v0.4.32` tag 自动构建并推送 `ghcr.io/hotll233/workload-tool:0.4.32` 与 `latest`
